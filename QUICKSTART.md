# Tech Blog Writer Skill - 快速开始（交互式版本）

## 🚀 一键启动

```bash
# 安装后直接运行
/tech-blog-writer

# 或指定路径运行
python3 ~/.claude/skills/tech-blog-writer/tech-blog-writer.py
```

## 🤖 当前配置

### Writer 模型：GLM-5.1

- **最大输出**：40000 tokens（约 3-4 万字）
- **成本**：~$0.15-0.30/篇（5000-10000字）
- **优势**：中文质量优秀、长文本连贯性好

### 模型选项

| 选项 | 说明 | 推荐场景 |
|------|------|----------|
| **专用模型** | 使用配置的 GLM-5.1 | 所有场景（推荐） |
| **主 Agent 模型** | 使用当前会话模型 | 无配置环境 |

## 📝 8步完成一篇文章

### 步骤1: 选择工作模式

```
请选择本次写作的工作模式：

  1. 自动模式 - 从网络自动获取最新技术文章候选
  2. 手动模式 - 手动指定已有的文章或主题

请选择 [自动模式/手动模式]: 1
```

### 步骤2: 选择文章

**自动模式**会显示候选列表：
```
╭────────────────────────────────────────╮
│ 候选文章列表                           │
├────┬──────────────┬────────┬──────┬───┤
│序号│ 标题         │ 来源   │ 评分 │...│
├────┼──────────────┼────────┼──────┼───┤
│ 1  │ STAR: LLM... │ arXiv  │ 4.5  │...│
│ 2  │ Transform... │ arXiv  │ 4.2  │...│
╰────┴──────────────┴────────┴──────┴───╯

请选择 [1-5]: 2
```

**手动模式**会提示输入：
```
请输入您想写作的主题信息：

文章标题: Transformer架构演进综述
参考链接（可选）: https://arxiv.org/abs/xxx
主题简介:
综述Transformer的发展历程，对比主流变体的优劣
（输入空行结束）
```

### 步骤3: 选择风格

```
请选择文章风格：

  1. 知乎专栏风
     特点：强烈第一人称、主观判断、个人经验分享

  2. 论文风
     特点：客观中立、引用真实案例、严谨表述

请选择 [知乎专栏风/论文风]: 2
```

### 步骤4: 生成初稿

```
正在生成文章初稿，请稍候...
  - 提取事实...
  - 生成大纲...
  - 生成骨架...
  - 写正文...

✓ 初稿已生成: pipeline/2026-05-19/draft.md
```

### 步骤5: 审阅修改

```
文章预览：
╭────────────────────────────────────────╮
│ # Transformer架构演进综述              │
│                                         │
│ Transformer架构通过自注意力机制...     │
╰────────────────────────────────────────╯

对初稿是否满意？ [Y/n]: Y
```

如果选择不满意：
```
请选择修改方式：
  1. 手动编辑文件
  2. 重新生成
  3. 继续下一步（稍后修改）

请选择 [手动编辑/重新生成/继续下一步]: 1

请在编辑器中打开文件进行修改：
  pipeline/2026-05-19/draft.md

修改完成后按回车继续...
```

### 步骤6: 添加图表

```
是否需要添加架构图、流程图或时序图？ [Y/n]: Y

请选择要添加的图表类型：
  1. 流程图 (flowchart)
  2. 时序图 (sequence)
  3. 甘特图 (gantt)
  4. 自定义 mermaid 图表
  5. 跳过

请选择 [流程图/时序图/甘特图/自定义/跳过]: 1

将添加 流程图...
提示：图表将自动插入到文章中合适的位置
```

### 步骤7: 选择发布渠道

```
请选择要发布的平台（可多选）：

  1. 掘金
  2. 知乎
  3. 微信公众号
  4. 个人博客
  5. GitHub
  6. 其他

请选择发布渠道（输入数字，多个用逗号分隔）: 1,2,3

已选择: 掘金, 知乎, 微信公众号
```

### 步骤8: 发布

```
文章路径: pipeline/2026-05-19/post.md
发布渠道: 掘金, 知乎, 微信公众号

正在发布到 掘金...
  提示：请手动复制到掘金编辑器发布
正在发布到 知乎...
  提示：请手动复制到知乎专栏发布
正在发布到 微信公众号...
  提示：请使用微信公众号编辑器发布

╭────────────────────────────────────────╮
│ 🎉 文章发布完成！                      │
╰────────────────────────────────────────╯

✓ 文章路径: pipeline/2026-05-19/post.md
✓ 发布渠道: 掘金, 知乎, 微信公众号
```

## ❓ 常见问题

### Q: 可以跳过某个步骤吗？

A: 部分步骤可以跳过：
- 步骤6（添加图表）：可以跳过
- 步骤8（立即发布）：可以稍后发布

### Q: 生成的初稿不满意怎么办？

A: 步骤5提供3种选择：
1. 手动编辑文件
2. 重新生成
3. 继续下一步（稍后修改）

### Q: 可以同时发布到多个平台吗？

A: 可以！步骤7支持多选，用逗号分隔，如：`1,2,3`

### Q: 如何安装美化界面？

A: 安装 rich 库：
```bash
pip install rich
```
