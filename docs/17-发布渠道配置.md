# 发布渠道配置

## 支持的发布渠道

### 1. 掘金
- **类型**: 技术社区
- **格式**: Markdown
- **特点**: 支持代码高亮、流量较大
- **发布方式**: 手动复制到编辑器
- **适合**: 技术教程、经验分享

### 2. 知乎
- **类型**: 知识问答社区
- **格式**: Markdown / 富文本
- **特点**: 支持长文、专栏功能
- **发布方式**: 手动复制到专栏
- **适合**: 深度分析、经验总结

### 3. 微信公众号
- **类型**: 自媒体平台
- **格式**: 富文本（需转换）
- **特点**: 移动端阅读、排版要求高
- **发布方式**: 使用编辑器（如 mdnice）转换后发布
- **适合**: 大众化内容、图文并茂

### 4. 个人博客
- **类型**: 个人网站
- **格式**: Markdown / HTML
- **特点**: 完全自主、SEO友好
- **发布方式**: 部署到博客系统（Hexo/Hugo等）
- **适合**: 所有类型内容

### 5. GitHub
- **类型**: 代码托管平台
- **格式**: Markdown
- **特点**: 版本控制、开源友好
- **发布方式**: 提交到仓库
- **适合**: 技术文档、开源项目

### 6. 其他平台
- **CSDN**: 国内技术社区
- **博客园**: 国内技术博客
- **SegmentFault**: 国内技术问答
- **Medium**: 国际平台（英文）
- **Dev.to**: 开发者社区

## 发布配置文件

```yaml
# publish_config.yaml
channels:
  - name: 掘金
    enabled: true
    format: markdown
    auto_publish: false
    url: https://juejin.cn

  - name: 知乎
    enabled: true
    format: markdown
    auto_publish: false
    url: https://zhuanlan.zhihu.com

  - name: 微信公众号
    enabled: true
    format: html
    auto_publish: false
    converter: mdnice  # 使用 mdnice 转换

  - name: 个人博客
    enabled: true
    format: markdown
    auto_publish: true
    deploy_command: "hugo && hugo deploy"
```

## 自动发布配置（可选）

如果需要自动发布，需要配置各平台的 API：

### 微信公众号
```bash
# 需要安装 wechatpy
pip install wechatpy

# 配置环境变量
export WECHAT_APP_ID="your_app_id"
export WECHAT_APP_SECRET="your_app_secret"
```

### GitHub
```bash
# 配置 GitHub Token
export GITHUB_TOKEN="your_token"
```

### 个人博客（Hugo）
```bash
# 配置部署命令
export BLOG_DEPLOY_CMD="hugo && aws s3 sync public/ s3://your-bucket"
```

## 发布流程

1. **生成文章** → `post.md`
2. **选择渠道** → 从配置中选择
3. **格式转换** → 根据渠道要求转换
4. **预览检查** → 人工确认
5. **发布** → 手动或自动发布

## 发布建议

### 知乎专栏风文章
推荐渠道：
1. 知乎专栏（首选）
2. 掘金
3. 微信公众号（需要转换排版）

### 论文风文章
推荐渠道：
1. 个人博客（首选）
2. GitHub
3. 知乎专栏

### 快速发布脚本

```bash
#!/bin/bash
# 快速发布到多个平台

DATE=$1
POST_PATH="pipeline/$DATE/post.md"

# 复制到剪贴板（macOS）
cat $POST_PATH | pbcopy
echo "文章已复制到剪贴板，请粘贴到目标平台发布"

# 或者使用自动化工具
# python scripts/auto_publish.py --date $DATE --channels juejin,zhihu
```

## 注意事项

1. **版权问题**: 确保文章内容原创或已获授权
2. **平台规则**: 遵守各平台的发布规范
3. **排版优化**: 不同平台可能需要调整排版
4. **发布时间**: 建议在用户活跃时段发布
5. **互动维护**: 发布后及时回复评论
