# 模型配置说明

## 当前配置

### Writer 模型：GLM-5.1

```yaml
writer:
  provider: jd_openai
  name: GLM-5.1
  temperature: 0.8
  max_tokens: 40000  # 支持 3-4 万字长文
  price_in_per_m:  20
  price_out_per_m: 80
```

### GLM-5.1 优势

| 特性 | 说明 |
|------|------|
| **最大输出** | 40000 tokens ≈ 3-4 万字 |
| **成本** | 输入 $20/M tokens，输出 $80/M tokens |
| **适合场景** | 长文写作、深度分析、论文风文章 |
| **特点** | 中文能力强、长文本连贯性好 |

### 成本估算

| 文章长度 | 估算 tokens | 成本估算 |
|----------|-------------|----------|
| 短文（2000字） | ~3000 tokens | ~$0.06 |
| 中文（5000字） | ~7500 tokens | ~$0.15 |
| 长文（10000字） | ~15000 tokens | ~$0.30 |
| 超长文（20000字） | ~30000 tokens | ~$0.60 |

## 使用建议

### 1. 长文写作

GLM-5.1 的 40000 tokens 输出能力，适合：
- ✅ 深度技术分析（10000-20000字）
- ✅ 论文风格文章
- ✅ 综述类文章
- ✅ 教程类文章

GLM-5.1 在成本和长度上都有优势。

## 性能优化建议

### 1. 根据文章长度选择 max_tokens

```python
# 短文（<5000字）
target_length = 3000
max_tokens = 5000

# 中文（5000-10000字）
target_length = 8000
max_tokens = 12000

# 长文（10000-20000字）
target_length = 15000
max_tokens = 25000

# 超长文（>20000字）
target_length = 25000
max_tokens = 40000
```

### 2. 分段生成（超长文）

对于超过 20000 字的文章，建议分段生成：

```yaml
# 在 brief.yaml 中设置
chosen:
  target_length: 30000
  split_strategy: "chapters"  # 按章节分段
```

### 3. 温度调整

```yaml
# 论文风（更严谨）
writer:
  temperature: 0.6

# 知乎风（更有个性）
writer:
  temperature: 0.9
```

## 故障排查

### 问题：文章被截断

**原因**：max_tokens 设置过小

**解决**：
```yaml
writer:
  max_tokens: 40000  # 增加到最大
```

### 问题：生成质量不稳定

**原因**：temperature 设置不当

**解决**：
```yaml
writer:
  temperature: 0.7  # 降低温度，提高稳定性
```

### 问题：成本过高

**原因**：长文消耗 tokens 较多

**解决**：
```yaml
# 方案1：限制文章长度
chosen:
  target_length: 5000

# 方案2：使用更便宜的模型
llm_by_stage:
  draft: reasoner  # 改用 DeepSeek
```
