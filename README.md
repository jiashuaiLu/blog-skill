# Tech Blog Writer Skill - 交互式工作流版本

专业技术博客写作助手，**完全交互式引导**，每一步都有明确提示。

## ✨ 核心特性

### 🎯 完全交互式

每一步都引导用户做出选择，无需记忆命令：

1. **步骤1**: 选择工作模式（自动/手动）
2. **步骤2**: 选择文章或主题
3. **步骤3**: 选择文章风格
4. **步骤4**: 生成初稿
5. **步骤5**: 人工审阅和修改
6. **步骤6**: 添加架构图/流程图
7. **步骤7**: 选择发布渠道
8. **步骤8**: 发布文章

### 🤖 模型配置

### 当前 Writer 模型：GLM-5.1

```yaml
writer:
  provider: jd_openai
  name: GLM-5.1
  temperature: 0.8
  max_tokens: 40000  # 支持 3-4 万字长文
```

### GLM-5.1 优势

| 特性 | 说明 |
|------|------|
| **最大输出** | 40000 tokens ≈ 3-4 万字 |
| **中文质量** | ⭐⭐⭐⭐⭐ |
| **长文本连贯性** | ⭐⭐⭐⭐⭐ |
| **成本** | $20/M (输入) + $80/M (输出) |

### 成本估算

| 文章长度 | 成本估算 |
|----------|----------|
| 短文（2000字） | ~$0.06 |
| 中文（5000字） | ~$0.15 |
| 长文（10000字） | ~$0.30 |
| 超长文（20000字） | ~$0.60 |

### 模型分配

| 阶段 | 模型 | 说明 |
|------|------|------|
| 粗筛 | Doubao-Seed-2-0-lite | 标题打分 |
| 推理 | DeepSeek-V4-Flash | 大纲生成 |
| 写作 | GLM-5.1 | 正文生成（40000 tokens）⭐ |

## 🎨 双风格支持

- **知乎专栏风**: 强烈第一人称、主观判断、个人经验分享
- **论文风**: 客观中立、引用真实案例、严谨表述

### 🔄 智能工作流

- **自动模式**: 从网络自动获取最新技术文章候选
- **手动模式**: 手动指定已有的文章或主题

## 📋 8步工作流详解

### 步骤1: 选择工作模式

| 模式 | 说明 | 适合场景 |
|------|------|----------|
| **自动模式** | 从网络自动获取最新技术文章候选 | 没有明确主题，想看看最新技术动态 |
| **手动模式** | 手动指定已有的文章或主题 | 已有明确的写作主题或参考资料 |

### 步骤2: 选择文章或主题

**自动模式**：展示候选文章列表（标题、来源、评分、摘要），用户选择感兴趣的文章

**手动模式**：手动输入：
- 文章标题
- 参考链接（可选）
- 主题简介

### 步骤3: 选择文章风格

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| **知乎专栏风** | 强烈第一人称、主观判断、个人经验分享 | 有个人实践经验、分享踩坑经历 |
| **论文风** | 客观中立、引用真实案例、严谨表述 | 技术深度分析、方法对比 |

### 步骤4: 生成文章初稿

自动执行：
1. 提取事实
2. 生成大纲
3. 生成骨架
4. 写正文

### 步骤5: 人工审阅和修改

选项：
1. 满意 → 继续下一步
2. 手动编辑文件
3. 重新生成
4. 继续下一步（稍后修改）

### 步骤6: 添加架构图/流程图

支持的图表类型：
- 流程图 (flowchart)
- 时序图 (sequence)
- 甘特图 (gantt)
- 自定义 mermaid 图表
- 跳过

### 步骤7: 选择发布渠道

支持的平台：
- 掘金
- 知乎
- 微信公众号
- 个人博客
- GitHub
- 其他

可多选，用逗号分隔。

### 步骤8: 发布文章

根据选择的渠道，提供发布指导：
- 手动发布：提供文章路径和发布提示
- 自动发布：（可选）集成平台 API

## 💡 使用建议

1. **首次使用**：建议选择自动模式，体验完整流程
2. **有明确主题**：使用手动模式，直接指定主题
3. **论文风格**：记得使用案例查询功能（步骤3会提示）
4. **发布前检查**：仔细审阅步骤5的文章预览
5. **图表补充**：步骤6可以为文章添加可视化内容
